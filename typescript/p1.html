Learn Typescript As Your First Programming Language

This guide is meant for those with no prior programming experience. It is not intended as a guide for those that already know Javascript or another language. This guide will teach you how to program. Each section will include practice problems and FAQs.

Introduction

What is typescript?
Typescript is a programming language which is a superset of Javascript. What this means is its basically Javascript with a few extra features.

Why typescript?
Many guides use python as a starting language due to its simplicity. However, I find it difficult for those learning to transition to other languages due to some weird / over simplified syntax. Some will insist Java is a good starting language. Java is more complicated but will help you understand many concepts after the steep learning curve. Typescript in my opinion is the best in between. It has a smaller learning curve, but it is very consistent and can be translated to Javascript or even other languages easily.

Setup

In order to get you started quickly, we will simply code in an online editor. This will let you edit and run your code without any setup. Simply open https://www.typescriptlang.org/play. Here you can edit code and hit the run button when you want to see the output. I suggest you copy and run most code examples. Every code example here can be indepdently run (unless noted) to make your life easier. Also try playing around with editing the examples if youre curious about certain changes. Lastly, note that the execises in each section are designed to be done in order.

Section 1: Variables and Basic Operations

1.1 Your first variable
Programming is all about keeping track of and manipulating information/data. The first basic concept in all programming languages is variables. A variable is simply some data that you assigned to a specific name. Heres an example

let myFavoriteColor = blue;

Now lets break this down.

What were doing here is created a variable named myFavoriteColor. We can name variables anything we want, whatever works to describe the data. Variable names cannot have spaces. In typescript, we typically use camel case for variable names, which means any new words in the name will start with a capital letter (other than the first word).

In order to declare this variable, we use the keyword let before the name.

For our variable, were assigning it a value of blue. We use an equal sign here. We can set the variable to be some number, or we can set the variable to some words / characters. When we use words / characters as our data, we call this a string (string of characters). Strings are always surrounded by quotes. These can be single or double quotes, typically sticking with single quotes unless our string contains an apostrophe. The variable name itself does not go in quotes, only the string were assigning to the variable.

Lastly, we end in a semicolon. This is used to denote the end of statement in code, similar to a period in a sentence. Well better understsand when to use these as we continue in this guide.

1.2 More on variable basics
After assign a variable, we can also reassign it. In this case, we dont need the let keyword, as the variable name has already been declared.

let myFavoriteColor = blue;
myFavoriteColor = red;

Now, anytime we reference our variable (more on referencing / using variables later) after reassigning it, it will have its new value.

Perhaps were sure our variable should not be changing. Perhaps we know our favorite color is blue and that wont change. We can actually declare a variable as a constant, and then typescript wont let us reassign the variable. We simply use const instead of let when declaring the variable. Here are some examples

const myFavoriteColor = 'blue';

const myLuckyNumber = 7;

If we tried reassigning the variable on the next line or anywhere in the code, typescript would give us an error. Besides this, the variables behave the same.

1.3 Types
So far, weve assigned variables that are either string or number. These are data types. In typescript, we can directly specify what type each variable is. In our past examples, we didnt specify a type, which means typescript will just do its best to figure it out. Heres an example where we specify a type

let myFavoriteColor: string = 'blue';

Here, after the variable name, we specify the type (after a colon). What this does is tell typescript that myFavoriteColor is a string and not anything else. This means we will get an error if we tried to reassign it to a number on the next line.

Heres another example

let myFavoriteNumber: number = 7;

Ill use this moment to introduce another type, which is called boolean. This just means true or false. Heres an example

let someCondition: boolean = true;

Specifying types explicitly may seem unnecessary now, but it is extremely useful for helping us organize our data, especially as our program becomes bigger and introduces more complex concepts.

1.4 Using Variables / basic operations
So far we have assigned some variables, but we havent actually used them. Before we get into any more complicated programming operations, lets learn how to use our variables for basic maths operations. Heres an example

const catAge: number = 12;
let dogAge: number = 10;
const totalAge: number = catAge + dogAge;

In this example, we have 2 different variables catAge and dogAge which each describe a pets age. If we want to calculate the total age, we can simply add the cats age and the dogs age together.

To do this, we reference the variables we assigned previously by their names. We dont put quotes around the variable name. If we were to accidentally put a variable name in quotes, typescript would not know were trying to use a variable, and would use the literal string we typed in the quotes.

We then add the 2 variables together using +. We assign this value to a new variable called totalAge. Whats important to note here as that everything on the right side of the assignment (right side of the =) is evaluated first before assigning that value to the variable. Therefore, totalAge will end up with the correct value of 22.

Once totalAge is assigned, the value is set and no longer depends on the other 2 variables. This means we can change the value of dogAge after on a 4th line, without actually affecting totalAge. In the following example, totalAge has a value of 22, even after line 4. While dogAge has a value of 11 after line 4.

const catAge: number = 12;
let dogAge: number = 10;
let totalAge: number = catAge + dogAge;
dogAge = 11;

If we wanted to change totalAge, we would have to reassign it.

const catAge: number = 12;
let dogAge: number = 10;
let totalAge: number = catAge + dogAge;
dogAge = 11;
totalAge = catAge + dogAge;

We can also use other math operations such as -, *, or /. These are for subtraction, multiplication or division respectively.

Lastly, we can also reassign variables based on their old values. Heres an example

let myFavoriteNumber: number = 10;
myFavoriteNumber = myFavoriteNumber + 1;

On the second line, were assigning the value of myFavoriteNumber to be the current value of myFavoriteNumber plus one. Remember, the right side of the = gets evaluated first. This means myFavoriteNumber + 1 must be evaluated first. This is the same as 10 + 1, which is 11. That means, the second line is reassigning myFavoriteNumber to be 11.

There is also a shortened syntax version of this that means the same thing

let myFavoriteNumber: number = 10;
myFavoriteNumber += 1;

1.5 Comments
If you want to take notes in your code, any line that begins with a // will be a comment. Comments do not affect the way your code runs, they are like taking notes. Here is an example:

// Here is how to assign a variable
let myFavoriteNumber: number = 10;

Section 2: Functions

2.1 Your first function
A function is a reusable piece of code, that takes in some input, does some work, and then gives back some output.

Lets say part of our code involved computing the average of 2 numbers. This code is nothing we havent done before:

const a: number = 10;
const b: number = 20;
const average: number = (a + b) / 2;

However, imagine you had to calculate the average, or even some more difficult computation, over and over again in your code. Youd end up repeating the same logic. Instead we can write a function that can be reused. Before going into a code example, lets understand our goal clearer. We want some reusable code which would have an input of 2 numbers, and then an output of a single number representing the average. Lets take a look

function computeAverage(a: number, b: number): number {
    const output = (a + b) / 2;
    return output;
}

This is a big jump in complexity so lets break it down carefully.

First, the keyword function is used to say we want to define a function. Next, the function name, in our case computeAverage.

The next section in parenthesis is our function input, or the function arguments. As described earlier, we want an input of 2 numbers. When defining a function, we need to give each argument a name, which is used for reference within the function. In this case, were just naming our arguments first and second. We use the colon and type notation from earlier to state that the arguments are numbers. We also separate each argument by a comma.

After the parenthesis, we have another : number which is specifying the return type (the data type of the output) of the function. In this case, our function is computing an average, which is going to be a number.

Next, inside curly braces, we have the body of the function. Inside the body, we can write any code we want to calculate the output of the function. We can reference our arguments by the names we gave them. In this case we compute an average as we did before. Once we have an output calculated, we use the keyword return to return the output, ending the function and finalizing the output.

You will notice that our function itself ends in a closing curly brace, but not a semi colon. Function definitions do not have semicolons at the end. A general rule is that if youre opening or closing a curly brace, you dont need a semicolon. You only need semicolons on individual statements.

2.2 Using functions
Now that weve written the function, we want to be able to use it. In our previous example, we had the following code:

const a: number = 10;
const b: number = 20;
const average: number = (a + b) / 2;

Now, with our function defined, we can use it. We call/invoke the function by writing the name of the function, followed by parenthesis containing the arguments were passing to the function. Heres our example now:

function computeAverage(first: number, second: number): number {
    const output = (first + second) / 2;
    return output;
}
const a: number = 10;
const b: number = 20;
const average: number = computeAverage(a, b);

What we do here is pass the values a and b to our computeAverage function. This gets evaluated. If we follow along with the function, we can see that it returns 15. This return value then gets assigned to our variable average.

You may be thinking that this is way more code to accomplish the same thing. However, imagine if you had to calculate the average of a bunch of different pairs of numbers. Or imagine if you had something more complex than calculating an average.

function computeAverage(first: number, second: number): number {
    const output = (first + second) / 2;
    return output;
}
const averageOne: number = computeAverage(10, 20);
const averageTwo: number = computeAverage(20, 20);
const averageThree: number = computeAverage(100, 200);

Note: we can call a function by directly providing values to it: computeAverage(10, 20) instead of assgining variables (a and b)to pass to it.

2.3 Console Output
So weve learned how to assign variables, and how to write a function. However, how do we actually know our function is doing what we want? When we run the code we have so far, we dont actually see anything on the screen. Typescript provides us a built in function that allows us to log to the console (the right side of the screen on https://www.typescriptlang.org/play). Heres an example

console.log('Hello World');

The function is called console.log and it can take a string as input. The function actually has no output, so you cant assign the output back to a variable like weve done previously. Instead, part of the function is logging/printing the input to the console. Run the code and you will see Hello World on the screen.

Going back to our earlier example, we can see the output of our function on the screen by doing this

function computeAverage(first: number, second: number): number {
    const output = (first + second) / 2;
    return output;
}
const a: number = 10;
const b: number = 20;
const average: number = computeAverage(a, b);
console.log(average);

Running this, we should see 15 on the screen, as thats the expected average of 10 and 20.

2.4 Understanding functions better: Argument Names
Lets look at our previous example again

function computeAverage(first: number, second: number): number {
    const output = (first + second) / 2;
    return output;
}
const a: number = 10;
const b: number = 20;
const average: number = computeAverage(a, b);

In this example, we assign values to 2 variables: a and b. After this, we call the function computeAverage with a and b. In our function definition, our 2 arguments are named first and second. You may have noticed the variable name mismatch. This is because when we define our function, we name the arguments simply to be able to reference them within our function body. The arguments dont have specific values yet and relying on the order of the arguments would be harder to follow. We give the names first and second because those are the names we want to use within our function. The values we pass to the function when we call it on the last line, get assigned to first and second as the output gets computed. As mentioned previously, we can call the function with either variables weve defined already or just with values directly: computeAverage(10, 20). Note: first and second are not accessible outside of the function

2.4 Understanding functions better: Variable Scope
The scope of a variable is essentially the area of code we can use the variable in. When we use let or const to declare a variable, this variable is scoped by the curly braces surrounding it. This means in our previous example, the variable output is not accessible outside of the function. This makes sense since output depends what first and second arguments are passed to the function.

If we have a variable defined outside of any curly braces, such as myFavoriteNumber, then this is accessible anywhere after its declaration in our code.

Section 3: If statements

3.1 What is an if statement?
So far, all our code can only do basic math operations. However, we want to be able to write much more complicated logic. For example, we may want to compute values differently based on a condition. In order to evaluate a condition, we use an if statement. Heres an example

function absoluteValue(x: number): number {
    if (x < 0) {
        return x * -1;
    }
    return x;
}

Lets break this down

First, we have a function definition for absoluteValue This function takes in a number, and returns a number. The goal of the function is to give us back the absolute value of the input (x). We know that if x is a posiitve number, then the absolute value of x is just x. If its negative, then its x multiplied by -1.

Lets look at our if statement. We begin by writing the word if, followed by a condition in parenthesis. The condition in this case is just checking if x < 0, in other words, if x is negative. More details on conditions are in the next subsection.

After the condition, we open curly braces to write the body of the if statement, similar to a function body. In this case, if the condition is true, we execute the code in the body of the if statement. If the condition is false, we do not execute the code. After the body / after the if statement, we simply continue with our code.

If you remember, when we return in a function, it ends the function right away and provides that value as output. That means in this case, if the condition is true, we return x * -1; and end the function. We never get to the second return statement if the condition is true. If the condition is false, we skip over the if statement body and execute the next line, return x;. In the end, we have a function that completes our goal of returning the absolute value of a number.

3.2 Conditions for if statements
For the condition, we can do some basic number comparisons using >, <, >= (greater or equal to), <= (less than or equal to), != (not equal to) or == (equal to). Note, for comparison, we use == (a double equals sign) because the single equals sign is meant for assignment of variables.

Code is made up of a bunch of building blocks. The condition for an if statement is actually just code that evaluates to true or false, which means its just a boolean. We can similarly assign booleans to variables, and even use them as our condition

const myNumber: number = 11;
const myNumberIsGreaterThanTen: boolean = myNumber > 10;

if (myNumberIsGreaterThanTen) {
    console.log('My number is greater than 10!');
}

We can also use true or false directly, though its not too useful for if statements

if (true) {
    console.log('This log statement will always happen!');
}

3.3 else if
We might want to do more complicated logic for conditions. We can combine if statements, with else if statements. Heres an example that Ill break down

const myNumber: number = 105;
if (myNumber > 100) {
    console.log('My number is greater than 100');
} else if (myNumber > 10) {
    console.log('My number is greater than 10');
}

Lets break it down.

First, we have a regular if statement, that says if myNumber is greater than 100, log a message stating its over 100.

Next, we have an else if statement. This is saying otherwise if myNumber is greater than 10, log a message saying its over 10. The else if is connected to the first if statement in that it only gets evaluated if the first if statement was false. In other words, if the number is 105, we dont check the else if condition, since the first condition was true.

This means at the end of our code example, weve only logged the first message, and not the second, even though the number is greater than both 100 and 10. If we wanted to log both messages, we simply wouldnt use an else if and just have 2 separate regular if statements like:

const myNumber: number = 105;
if (myNumber > 100) {
    console.log('My number is greater than 100');
} 
if (myNumber > 10) {
    console.log('My number is greater than 10');
}

If myNumber was 50, in both examples wed only log the second message.

To summarize, the else if statement gets evaluated only when the if statement before it was false. Hence, I like to teach it as an otherwise if statement. If condition A, execute body A, otherwise if condition B, execute body B.

Note: you always need an if statement before an else if. Without the first condition, the else part doesnt have any meaning.

Note: the spacing here doesnt matter. The else if in our first example is on the same line as the closing brace for the first if statement. However, it can be on the next line and it will behave the same. We keep it on the same line typically so that its more clear these statements are connected. Heres an example with alternative spacing, but the same behavior

const myNumber: number = 105;
if (myNumber > 100) {
    console.log('My number is greater than 100');
}
else if (myNumber > 10) {
    console.log('My number is greater than 10');
}

3.4 Chaining else ifs

We can also continue to chain our if statements with more else if statements.

const myNumber: number = 60;
if (myNumber > 100) {
    console.log('My number is greater than 100');
} else if (myNumber > 50) {
    console.log('My number is greater than 50');
} else if (myNumber > 10) {
    console.log('My number is greater than 10');
}

In this example, we will only see the second message logged. The first condition is false, so we move on to checking the next one. This second condition is true, so we log the second message. However, since the second condition was true, we dont try to evaluate the third condition.

3.5 else

Lastly, we can include an else statement at the end of our chain. Well execute the else statements body if all the other conditions in the chain were false.

const myNumber: number = 6;
if (myNumber > 100) {
    console.log('My number is greater than 100');
} else if (myNumber > 50) {
    console.log('My number is greater than 50');
} else if (myNumber > 10) {
    console.log('My number is greater than 10');
} else {
    console.log('My number is smol');
}

This example will log the last message only.

The else statement does not need a condition, as it depends on all the other conditions in the chain.

Note: the else statement can be used with or without any else if statements. You only require the first statement.

const myNumber: number = 50;
if (myNumber > 100) {
    console.log('My number is greater than 100');
} else {
    console.log('My number is smol');
}

This example will still only log My number is smol.

Section 4: Loops

4.1 What is a while loop?
Often times, well need to repeat certain code over and over. In many cases, the number of times we repeat some code might be based on something else in the code. For something like this, we use a loop, to repeat code. A simple form of loops is a While Loop. In a while loop, we check a condition, if that condition is true, we run some code, then we check the condition again and potentially repeat. Thus we repeat code until the condition is no longer met. Heres an example well break down

let i: number = 1;
while (i <= 3) {
    console.log('Hello!');
    i = i + 1;
}

Lets break it down

Before the while loop, we define a variable i as weve done before

Then we have the while loop, which first begins with the keyword while, followed by a condition which works the same as an if statement, then curly braces and then the while loop body inside the curly braces.

When we execute this code, we first check the condition. If the condition is met, we execute the while loop body (inside the curly braces), similar to an if statement. The difference here is after we execute the body, we will check the condition again. If the condition is still true, well repeat the body again. This will continue in this cycle until the condition is false. Once its false, we have finished the while loop.

4.2 Breaking down the example further
Lets break this down even further by understanding what code actually gets executed.

First, we define a variable. Next we have a while loop condition. At this point, i is 1, while means our condition is true, since i <= 3. We then execute the while loop body

i condition check executed code
1 true console.log(Hello!); i = i + 1;

After we execute the body, weve completed our first iteration of the loop. The code that executed first logged to the console, and then it changed the value of i, incrementing it by 1. That means weve now ended our first iteration and i is 2. We then check our condition again, which is true so we execute the body again

i condition check executed code
1 true console.log(Hello!); i = i + 1;
2 true console.log(Hello!); i = i + 1;

After that second iteration, i will be 3, and we will have logged to the console again. Lets continue.

i condition check executed code
1 true console.log(Hello!); i = i + 1;
2 true console.log(Hello!); i = i + 1;
3 true console.log(Hello!); i = i + 1;

Okay, after this third iteration i, will be 4. At this point, the condition is false, so we dont execute the body.

i condition check executed code
1 true console.log(Hello!); i = i + 1;
2 true console.log(Hello!); i = i + 1;
3 true console.log(Hello!); i = i + 1;
4 false

Our while loop is now complete.

Note: without the line that increments i, we will have an infinite loop.

4.3 More examples
With while loop exercises, I like to think the biggest piece is figuring out which information we need to keep track. The next biggest pieces are when we need to stop (our while condition), and what code to repeat.

Lets say we wanted a function which took in a number, x and was supposed to return the sum of all the numbers up to and including x. In other words, if x was 5, our function would return 1 + 2 + 3 + 4 + 5, which is 15.

Lets start with a function header. We know our function takes in one number and returns a number

function sumUpTo(x: number): number {
    
}

Inside our function, we know we need a loop in order to repeat some math until we reach our sum. However, before we write the while loop, we need to track some information. The first piece of information we need to track outside the loop itself, is the current number were on (variable currentNumber). We should start from 0 first. For each iteration of the loop, we want to go through each number from 0 to x. So well use a conditon based on currentNumber and x, which will be currentNumber < x. Within our body, well want to make sure we go through each value of currentNumber one by one, so well increment it by 1.

function sumUpTo(x: number): number {
    let currentNumber: number = 0;
    while (currentNumber <= x) {
        currentNumber += 1;
    }
}

Now at this point we have a loop that goes through the numbers 0 to x. However, it doesnt actual compute anything. We know we need a sum of numbers. For each currentNumber, we want to add to a rolling sum, well call currentSum. currentSum should start from 0.

function sumUpTo(x: number): number {
    let currentNumber: number = 0;
    let currentSum: number = 0;
    while (currentNumber <= x) {
        currentSum = currentSum + currentNumber;
        currentNumber += 1;
    }
}

At the end of our loop, currentSum will be the sum of all the numbers 0 to x, so lets return it to complete our function.

Note, this final code snippet for section 4.3 is a complete code snippet:

    let currentSum: number = 0;
    while (currentNumber <= x) {

        currentNumber += 1;

    return currentSum;

Section 5: Arrays

5.1 What are arrays
In many cases, storing a single variable or a few variables does not truly represent the data youre trying to work with. Instead, you may need a variable that keeps track of a list of values. Thats what an Array is:

const myList = [1, 2, 3, 4, 5];

In this example, myList is an array of numbers. Its a single variable that stores multiple numbers, and in a specific order. You defined it like any other variable, writing the values in square brackets, each separated by a comma.

If you wanted to write the type, you can use Array followed by the type of each item in the array written in angled brackets. Here are some examples

const myListOfNumbers: Array<number> = [1, 2, 3, 4, 5];
const myListOfString: Array<string> = ["david", "lisa"];

5.2 How to access values arrays
Lets say we have an array defined:

const myFavoriteNumbers = [500, 32, 9];

The values in the array are ordered. Each value has an index that can be used to reference it. For arrays in typescript, the indices begin at 0. That means the first value in the array is at index 0. At index 1, we have the second value of the array.

We can specifically access the values by their index in code, by writing the variable name, followed by the index in brackets:

const myFavoriteNumbers: Array<number> = [500, 32, 9];
const myFirstFavoriteNumber: number = myFavoriteNumbers[0];

In this code, myFirstFavoriteNumber is going to be 500. We can change the 0 on the second line to be a different index if wed like to access a different value.

5.3 Array properties and methods
As of now, we havent really seen anything cool we can do with arrays that we cant do with a bunch of variables.

Arrays have properties we can access, besides just their values by index. The most common thing we might want is to access the length of an array

const myFavoriteNumbers: Array<number> = [500, 32, 9];
const howManyNumbers: number = myFavoriteNumbers.length

Here, howManyNumbers will be assigned to 3 because there are 3 numbers in myFavoriteNumbers.

Arrays also have methods. These are like functions, except they are attached to the actual array instance. A simple example is the push. This is a method thats meant to add to the array.

const myFavoriteNumbers: Array<number> = [500, 32, 9];
myFavoriteNumbers.push(7);

After calling the push method, the array will have a 4th value, which is 7.

This introduces a new concept of mutation. You may have noticed we used const here yet the array can still change. In this case, were not reassigning the variable. Instead, were directly mutating the values inside the array. In other words, myFavoriteNumbers is still the same array, but we added to the end of it. You can see we dont use the = to reassign the variable.

For the purposes of this tutorial, were going to stick with just the length property and the push method. You can find a lot more methods and details here if youre interested: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array

5.4 Combining loops and arrays
We can truly do a lot more now that we know about arrays. Particularly, by combining them with loops, we can do a lot of different computations. Lets look a previous example from section 4:

function sumUpTo(x: number): number {
    let currentNumber: number = 0;
    let currentSum: number = 0;
    while (currentNumber <= x) {
        currentSum = currentSum + currentNumber;
        currentNumber += 1;
    }
    return currentSum;
}

This function calculated the sum of all the numbers from 1 to x. This is cool, but we can make more useful functions now. For example, lets say we wanted to calculate the sum of all the values in a given array. In our old example, we kept looping until currentNumber reached x. However, with an array, we dont want to rely on the contents. Instead, we want to make use of the length property. We know that in the array, indices 0 up to but not including the length exist. In other words, if we have a list of 3 numbers. We know that we can access the values via indices 0, 1, and 2. With this knowledge, we can take out previous function except instead of keeping track of currentNumber, were going to keep track of the currentIndex. Were going to keep looping until currentIndex reaches the length. Inside the loop body, well use currentIndex to actually access the value from the list. Heres our final function

function sumOfNumbersInArray(numbers: Array<number>): number {
    let currentIndex: number = 0;
    let currentSum: number = 0;
    while (currentIndex < numbers.length) {
        currentSum = currentSum + numbers[currentIndex];
        currentIndex += 1;
    }
    return currentSum;
}

Id encourage you to go through this example with table of variables as we did when learning about loops.

Many exercises including arrays will involve keeping track of an index like this. Always think about the relationship between the index, the value at that index, and the length of the array.
